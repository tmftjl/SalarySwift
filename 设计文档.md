# SalarySwift 设计文档

## 一、项目概述

SalarySwift 是一款面向小型作坊场景的本地化工资记录 App，当前目标平台为 Android。产品重点不是复杂报表，而是让单一使用者能在手机上快速完成以下动作：

- 管理员工名单
- 录入当期工资
- 一键归档当前批次
- 查看历史批次
- 导出单批次 PDF 并通过系统分享

当前版本不涉及登录、联网、权限分级和多端同步。

## 三、数据模型

### 3.1 employees 表

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PK AUTOINCREMENT | 主键 |
| `name` | TEXT NOT NULL | 员工姓名 |
| `is_active` | INTEGER DEFAULT 1 | 1=在职，0=软删除 |
| `created_at` | INTEGER | Unix 时间戳 |

说明：

- 软删除后员工不会再进入未来的待录入名单。
- 已归档历史记录仍保留员工名称和金额。
- 若员工存在当前 `active` 工资记录，删除时会同时移除该未归档记录，避免工作台出现看不见的数据。

### 3.2 salary_records 表

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PK AUTOINCREMENT | 主键 |
| `employee_id` | INTEGER FK | 关联 `employees.id` |
| `amount` | REAL | 工资金额 |
| `status` | TEXT | `active` / `settled` |
| `batch_key` | TEXT NULL | 结算批次，格式 `YYYY-MM` |
| `created_at` | INTEGER | 记录更新时间戳 |

说明：

- `active` 表示当前工作台中的未归档工资。
- `settled` 表示已经归档到某个历史批次。
- 当前模型只保存“某次结算的最终金额”，不保存“工资归属月份”。
- 系统会在写入 `active` 记录时按员工去重，避免同一员工出现多条未归档工资。

### 3.3 当前可支持的核心查询

```sql
-- 工作台：当前 active 记录
SELECT * FROM salary_records WHERE status = 'active';

-- 工作台：待录入员工
SELECT * FROM employees
WHERE is_active = 1
  AND id NOT IN (
    SELECT employee_id FROM salary_records WHERE status = 'active'
  );

-- 历史批次列表
SELECT batch_key, SUM(amount), COUNT(*)
FROM salary_records
WHERE status = 'settled'
GROUP BY batch_key
ORDER BY batch_key DESC;

-- 某批次详情
SELECT e.name, sr.amount
FROM salary_records sr
JOIN employees e ON sr.employee_id = e.id
WHERE sr.batch_key = '2026-03';
```

## 四、页面与交互

底部导航三页：**工作台 / 员工库 / 历史记录**

### 4.1 工作台

功能目标：

- 展示当前已录入总额
- 区分待录入和已录入员工
- 支持连续录入
- 支持导入最近一次批次作为初始值
- 支持结算归档

当前交互规则：

- 顶部展示当前 `active` 工资总额
- 待录入员工在列表中直接输入金额
- 键盘使用数字输入，回车后跳到下一位员工
- 已录入员工显示金额，并支持：
  - 编辑金额
  - 撤回录入，重新回到待录入
- `导入上月` 会把最近一个历史批次复制为当前 `active` 记录，导入后可在“已录入”区域继续修改
- `完成并归档` 只在所有在职员工都已录入时显示

### 4.2 员工库

功能目标：

- 新增员工
- 修改姓名
- 软删除员工

当前交互规则：

- 右上角按钮新增员工
- 点击员工卡片可修改姓名
- 左滑员工卡片可删除
- 删除为软删除，不影响已归档历史

### 4.3 历史记录

功能目标：

- 查看已归档批次
- 查看批次明细
- 导出单批次 PDF

当前交互规则：

- 历史列表按 `batch_key` 倒序展示
- 每项展示批次、人数和总金额
- 进入详情页后展示员工明细与合计金额
- 详情页底部支持导出并分享 PDF
- 历史页会自动监听数据变化，结算后无需重启页面即可看到新批次

## 五、PDF 导出设计

当前版本导出的是**单批次明细表**，不是跨月矩阵报表。

示例：

| 员工 | 金额 |
|------|------|
| 张三 | 5,000.00 |
| 李四 | 4,500.00 |
| 合计 | 9,500.00 |

实现说明：

1. 使用 `pdf` 生成 A4 明细页
2. 文件先写入应用临时目录
3. 通过 `share_plus` 打开系统分享面板
4. PDF 固定使用随 App 打包的 `assets/fonts/NotoSansSC-Regular.otf` 与 `assets/fonts/NotoSansSC-Bold.otf`

限制说明：

- 当前数据表没有“工资所属月份”字段，因此无法可靠生成“员工 x 月份”的交叉表
- 如果后续要做跨月矩阵 PDF，至少需要新增类似 `salary_month` 的字段，并重新定义批次与导出逻辑

## 六、业务流程

### 6.1 当期录入

```text
进入工作台
  ↓
拉取所有在职员工
  ↓
已存在 active 记录的进入“已录入”
没有 active 记录的进入“待录入”
  ↓
录入或编辑金额
  ↓
全部员工完成后，显示“完成并归档”
  ↓
确认归档
  ↓
所有 active 记录写入 settled，并标记 batch_key = 当前 YYYY-MM
```

### 6.2 导入最近批次

```text
点击“导入上月”
  ↓
读取最近一个 settled 批次
  ↓
按员工复制为当前 active 记录
  ↓
工作台自动刷新到“已录入”区域
  ↓
用户可继续编辑或撤回
```

### 6.3 删除员工

```text
员工库左滑删除
  ↓
员工 is_active = 0
  ↓
若该员工存在未归档 active 记录，则同步移除
  ↓
历史 settled 记录保持不变
```

## 八、当前 MVP 范围

### 已实现

- 底部三页导航
- 员工新增、改名、软删除
- 工作台待录入/已录入分组
- 连续金额录入
- 导入最近批次
- 已录入金额编辑与撤回
- 一键归档
- 历史批次列表与详情
- PDF 导出与系统分享

### 暂不包含

- 登录与多用户
- 云同步
- 数据备份恢复
- 跨月矩阵工资报表
- 多币种与统计图表

## 九、开发注意事项

- 当前仓库未包含 Drift 生成文件，首次开发前需执行代码生成。
- 当前仓库已包含 PDF 所需中文字体，并随 App 打包。
- 如果后续需要真正的“按月份展开工资历史”，应先升级数据模型，不应直接在现有 `batch_key` 上叠加复杂报表逻辑。
