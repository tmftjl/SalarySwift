# SalarySwift 设计文档

## 一、项目概述

SalarySwift 是一款本地化、极简操作的安卓端员工工资记录 App，专为小型作坊场景设计。核心目标：

- 解决手机端批量录入繁琐的痛点
- 防漏算提醒（智能分组提示未录入员工）
- 快速归档与 PDF 导出（可发微信/本地存档）

**目标用户**：单一非技术用户，无需登录、无需联网、无权限管理。

## 三、数据库设计

### 3.1 employees 表

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PK AUTOINCREMENT | 主键 |
| `name` | TEXT NOT NULL | 员工姓名 |
| `is_active` | INTEGER DEFAULT 1 | 1=在职，0=软删除 |
| `created_at` | INTEGER | Unix 时间戳 |

> **软删除**：删除员工只将 `is_active` 置为 0，历史工资记录完整保留。

### 3.2 salary_records 表

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PK AUTOINCREMENT | 主键 |
| `employee_id` | INTEGER FK | 关联 employees.id |
| `amount` | REAL | 工资金额 |
| `status` | TEXT | `active`（当前周期）/ `settled`（已归档） |
| `batch_key` | TEXT | 结算批次标识，格式 `YYYY-MM`，active 时为 NULL |
| `created_at` | INTEGER | Unix 时间戳 |

### 3.3 核心 SQL 逻辑

```sql
-- 工作台：已录入列表
SELECT * FROM salary_records WHERE status = 'active';

-- 工作台：待录入列表（在职员工中本月未录入的）
SELECT * FROM employees
WHERE is_active = 1
  AND id NOT IN (SELECT employee_id FROM salary_records WHERE status = 'active');

-- 历史批次列表
SELECT batch_key, SUM(amount), COUNT(*)
FROM salary_records
WHERE status = 'settled'
GROUP BY batch_key
ORDER BY batch_key DESC;

-- 批次明细
SELECT e.name, sr.amount
FROM salary_records sr
JOIN employees e ON sr.employee_id = e.id
WHERE sr.batch_key = '2026-03';
```

## 四、页面设计

底部导航三页：**工作台 / 员工库 / 历史记录**

### 4.1 工作台（当月结算页）

```
┌─────────────────────────────────┐
│  本月工资合计                    │
│  ¥ 28,500.00          [导入上月] │
├─────────────────────────────────┤
│ 🔴 待录入 (3人)                  │
│   张三          [____] ¥         │
│   李四          [____] ¥         │
│   王五          [____] ¥         │
├─────────────────────────────────┤
│ 🟢 已录入 (5人)                  │
│   赵六                  5,000 ¥  │
├─────────────────────────────────┤
│            [ 一键结算 ]           │
└─────────────────────────────────┘
```

**交互细节：**
- 顶部醒目展示当前录入总额（实时更新）
- 待录入组（🔴）：本月无 active 记录的在职员工，内联输入框
- 已录入组（🟢）：有 active 记录的员工，显示金额
- 键盘类型：纯数字键盘（`keyboardType = TextInputType.number`）
- IME Action 设为 `next`：按"下一步"自动跳到下一个待录入员工，无需收起键盘
- **导入上月**：一键复制最近一批 settled 记录为新 active 记录，预填金额供微调
- **一键结算**：弹确认框 → 确认后所有 active 记录写入 `status=settled`、`batch_key=当前YYYY-MM`，工作台清空

### 4.2 员工库（人员管理页）

```
┌─────────────────────────────────┐
│  员工库                  [+ 新增] │
├─────────────────────────────────┤
│  张三          ← 左滑显示删除按钮  │
│  李四                            │
│  王五                            │
└─────────────────────────────────┘
```

**交互细节：**
- 点击「+ 新增」：底部弹出 BottomSheet，输入姓名后保存
- 左滑：显示红色删除按钮，点击执行软删除（`is_active = 0`）
- 点击员工名：可编辑姓名

## 五、PDF 导出格式

采用**交叉表（矩阵）格式**：行为员工，列为月份，最右列为行合计，最下行为列合计。

**示例（2026年1月—3月批次）：**

| 员工 | 1月 | 2月 | 3月 | 合计 |
|------|-----|-----|-----|------|
| 张三 | 5,000 | 5,000 | 5,200 | 15,200 |
| 李四 | 4,500 | 4,500 | 4,500 | 13,500 |
| **合计** | **9,500** | **9,500** | **9,700** | **28,700** |

> 注：一个批次不一定只含一个自然月，从上次结算到本次结算之间的所有月份均作为列展示。

**技术实现：**
1. 用 Flutter `pdf` 包的 `pw` 组件树绘制表格
2. PDF 文件保存至应用 cache 目录
3. 通过 `share_plus` 弹出系统分享面板

## 六、业务流程

### 月度录入流程

```
新月份开始
    ↓
工作台自动拉取所有在职员工 → 全部进入"待录入"组
    ↓
【可选】点击「导入上月」→ 以上月金额预填
    ↓
逐人录入，按"下一步"跳至下一人
    ↓
全部录入完成 → 点击「一键结算」
    ↓
确认弹框 → 确认
    ↓
所有 active 记录：status = settled，batch_key = 当前 YYYY-MM
工作台清空，准备下月
```

### 结算批次说明

一个"批次"不等于一个自然月。批次由两次结算操作之间的时间跨度决定。例如：1月结算后，4月才再次结算，则4月批次包含1月+2月+3月+4月共四列数据。

### 软删除完整性保证

从员工库删除员工，仅隐藏其在未来工作台中的显示，历史批次中该员工的所有记录完整保留，PDF 导出正确呈现。

## 八、MVP 范围

### 包含

- 底部三页导航（工作台 / 员工库 / 历史记录）
- 员工新增 / 编辑 / 软删除
- 工资录入：智能分组（待录入 vs 已录入）、纯数字键盘、IME Next 连续跳转

- 「一键结算」含确认弹框

- 数据备份与恢复
- 多币种支持
- 统计图表
